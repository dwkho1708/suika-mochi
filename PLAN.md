## 3. 게임플레이 시스템

### 3.1 핵심 루프
```
NEXT 친구 확인 → 마우스/터치로 위치 조정 → 클릭/손가락 뗌으로 드롭
   ↓
물리 낙하 → 같은 친구와 닿으면 합치기 → 다음 단계로 진화
   ↓
점수 + 콤보 체크 → NEXT 갱신 (동적 확률) → 반복
   ↓
위험선 위에 일정 시간 머무는 친구 발생 → 게임오버
```

### 3.2 드롭 규칙
- 항상 통 위 일정 높이에서 떨어뜨림
- 드롭 후 다음 드롭까지 **0.6초 쿨다운** (연타 방지)
- **NEXT 동적 확률 풀** (통 점유율 = 디저트 총 질량 / 최대 수용 질량)
  - `fillRatio < 0.35` (초반): `[1,1,2,2,3,3,4,5]` 균등
  - `0.35 ≤ fillRatio < 0.65` (중반): `[1,1,1,2,2,3,3,4]` 작은 쪽 가중
  - `fillRatio ≥ 0.65` (후반): `[1,1,1,1,2,2,3]` 정리용 작은 것 위주

> 게임오버 직전 큰 NEXT가 떨어지는 불쾌감 방지

### 3.3 합치기 규칙
- 같은 단계 두 친구 충돌 → 두 개 제거, **두 바디의 질량 중심점**에 한 단계 위 친구 생성
- 동시 충돌 여러 쌍이면 한 프레임에 모두 처리
- **충격파 보정**: 생성 순간 주변 디저트에 미세 force 적용 (물리 꼬임 방지)
- **11→11 (3단 케이크 합치기)**: 사라지지 않고 보너스 점수 +300 + 화면 전체 파티클 세리머니
  - 같은 세션에서 N회 완성 시 "👑 케이크 마스터 x{N}" 토스트

### 3.4 게임오버
- **위험선**: 통 상단에서 70px 아래
- 친구의 **`body.bounds.min.y`**(최상단)가 위험선 위로 올라가서 **2초 이상** 머물면 게임오버
  - ❌ 중심점 기준은 큰 케이크가 통 밖으로 튀어나와도 게임오버 안 됨
- **드롭 직후 0.5초** 또는 **첫 충돌까지**는 위험선 판정 유예 (NEXT 낙하 중 즉시 OUT 버그 방지)
- 위험선 닿는 순간 빨강 점멸 경고

### 3.6 콤보 시스템 (v2 유동적 윈도우)

- 합치기 성공 후 일정 시간 안에 다음 합치기 → 콤보 +1
- **유동적 윈도우** (직전 머지 단계에 따라):

| 직전 머지 단계 | 다음 합치기 윈도우 |
|---|---|
| 1~5단계 | 1.5초 |
| 6~8단계 | **1.8초** |
| 9~10단계 | **2.0초** |
| 11단계 | **2.5초** |

- **점수 배율**: 1콤보=x1.0, 2=x1.2, 3=x1.5, 4=x1.8, 5+=x2.0, **8콤보 이상 cap x2.5**
- 화면 중앙 상단에 "콤보 x3!" 토스트 + 별 파티클

> 물리 연쇄 반응(체인 리액션) 시간 고려해 충분히 여유

## 6. 기술 설계

### 6.1 스택
- **HTML5 + Vanilla JavaScript** — 빌드 도구 없음, `index.html` 더블클릭 실행
- **Matter.js** — 2D 물리
- **Canvas 2D + 오프스크린 라스터라이징** — 11종 디저트를 게임 로딩 시 메모리 캔버스에 미리 그려두고 `drawImage` 블리팅
- **Howler.js** 또는 순수 `new Audio()` Pool — 오디오 (2차에서 도입)
- **localStorage** — 점수·설정 저장
- 외부 폰트: Google Fonts (Gowun Dodum, Quicksand)

```js
    return;
  }

  const newBody = createDessert(nextId, newPos.x, newPos.y);
  Matter.World.add(world, newBody);

  // 충격파: 주변 디저트에 미세 force (물리 꼬임 방지)
  nearbyBodies(newPos, 60).forEach(b => {
    const dir = Matter.Vector.normalise(Matter.Vector.sub(b.position, newPos));
    Matter.Body.applyForce(b, b.position, Matter.Vector.mult(dir, 0.0005));
  });

  spawnMergeParticles(newPos);
  addScore(DESSERTS[bodyA.dessertId - 1].mergeScore * comboMultiplier);
  bumpCombo(bodyA.dessertId);
}
```

### 6.7 위험선 판정 (v2 수정)
```js
function checkGameOver(dt) {
  for (const body of dynamicBodies) {
    // 드롭 직후 유예
    if (body.gracePeriod > 0) {
      body.gracePeriod -= dt;
      continue;
    }
    // 첫 충돌 전이면 면제
    if (!body.hasCollided) continue;

    // 중심점이 아닌 최상단 좌표 기준
    if (body.bounds.min.y < dangerY) {
      body.dangerTime += dt;
      if (body.dangerTime > 2000) return endGame();
    } else {
      body.dangerTime = 0;
    }
  }
}
```

### 6.8 오프스크린 라스터라이징 (Retina 대응)
```js
function prerenderSprites() {
  const dpr = window.devicePixelRatio || 1;
  for (const d of DESSERTS) {
    const size = d.radius * 2;
    const off = document.createElement('canvas');
    off.width = size * dpr;
    off.height = size * dpr;
    const offCtx = off.getContext('2d');
    offCtx.scale(dpr, dpr);
    drawDessertVector(offCtx, d.id, size); // SVG 기반 path 그리기
    spriteCache[d.id] = off;
  }
}

// 게임 루프에서는 단순 blit
function drawDessert(body) {
  const sprite = spriteCache[body.dessertId];
  const d = DESSERTS[body.dessertId - 1];
  ctx.save();
  ctx.translate(body.position.x, body.position.y);
  ctx.rotate(body.angle);
  ctx.drawImage(sprite, -d.radius, -d.radius, d.radius * 2, d.radius * 2);
  ctx.restore();
}
```

## 10. 위험 / 제약

### 10.1 기술적 위험
- **모바일 성능**: 11종 디저트 + 파티클 → 오프스크린 스프라이트 + 파티클 캡(60개) + 디저트 최대 80개 캡
- **물리 안정성**: 동시 다발 합치기 + 충격파 보정 + positionIterations 8
- **터널링**: 벽 두께 30px (모찌 반지름 18px의 1.7배), `slop: 0.05`
- **드롭 직후 게임오버 버그**: 0.5초 유예 + 첫 충돌 전 면제 (§6.7)
- **위험선 오판정**: `bounds.min.y` 기준으로 큰 케이크 튀어나옴 정확히 감지

### 10.2 디자인 위험
- 11종이 비슷한 톤이라 구분 어려움 가능 → 모양·크기·표정 차이 강조 (시안 반영 완료)
- 큰 화면에서 통이 작아 보임 → 최대 너비 제한 (게임 영역 420px 고정)

### 10.3 결정사항 (리뷰 반영 완료)
| # | 항목 | 결정 |
|---|---|---|
| 1 | 점수 곡선 | 8단계 이후 가중치(x1.4/x1.8/x2.4), 11→11 보너스 +300 |
